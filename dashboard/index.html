Order-based Integration

                            
Jodo sends real-time notifications for Pay Order events. This is the primary integration method for current sandbox testing.

                            
                            
Available Events

                            

                                
Example JSON

                                
"event": "order.payment.debited"
"data": {
    "id": "pay_8607c3cb94ae",
    "order_id": "order_fd8ebc8f1e27",
    "amount": 1500,
    "status": "paid"
}

                            

                            
    
                                
  •  order.payment.debited - Triggered when a Pay order payment is made.
    • 
                                
  •  order.payment.settled - Triggered when a Pay order payment is settled to the bank account.
    • 
                            

                        


                        
                        

                            
Reliability & Retries

                            
To ensure events are delivered successfully, Jodo implements a robust retry policy. We consider delivery successful only when your system returns an HTTP 200 OK.

                            

                                
                                

                                    Retry Policy: Jodo retries up to 5 times over a period of 3 days. Retries happen at 6-hour and 24-hour intervals.
                                

                            

                        


                        
                        

                            
Ensure Idempotency

                            
Due to the retry logic, your system might receive the same event multiple times. You should ensure that your application handles this gracefully without duplicate processing.

                            

                                
                                

                                    Best Practice: Always check the Order ID or Payment ID before updating your database. Return 200 OK even if the event was already processed.

Configure your firewall to allow traffic from the following IP addresses:

2. Signature Verification

                            
Use the X-Jodo-Signature header to verify the payload using your Webhook Secret and HMAC-SHA256.

                        


                        
                        

                            
Handling Failures

                            
If a webhook fails continuously (100 times in Production, 10 times in UAT), the subscription is automatically disabled to prevent system overload. You will receive an email notification for such events.
